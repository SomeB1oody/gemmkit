# 测试与验证

一个把“这里逐位一致、那里按容差、处处可复现”写进头条承诺的库，成败全在测试能多精确地钉住这些字眼。gemmkit 的测试套件位于 `gemmkit/tests/`，第一个结构性决定是什么*不算*测试：性能 harness 是测量工具，绝不是 CI 门禁。`tests/perf/` 是详尽的内部调查套件——`#[ignore]` 的基准测试跑在取 9 次中位数的 harness 上，因为每个基准都会打满所有核心而在一把共享锁后串行——在改动需要数据时手动运行。`gemmkit/benches/gemm_bench.rs` 是精选的公开 `cargo bench` 面：criterion 基准分五个头条组（`sgemm`、`dtypes`、`gemv`、`prepacked`、`batched`），用于对 `gemm` crate 和 `matrixmultiply` 做 `--save-baseline` 回归追踪。两者都不能让合并失败，因为共享 CI runner 上的性能断言断言的主要是噪声。

## 正确性、性质、一致性、模糊测试

正确性套件（`tests/correctness/`）以形状 x 布局 x alpha/beta 组合扫描，对照一个独立的 `f64` 参考 GEMM。参考实现及其精度机制只存在一份，在 `tests/oracle_common/` 里——元素 trait、确定性填充、`f64` 参考、每个元素类型的相对 Frobenius 精度门——由正确性和性质两套 harness 以 `#[path]` 包含，所以值得信任的 oracle 只有一个。oracle 扫描之上还有：对外部 `gemm` crate 的交叉核对（一个独立实现，能抓住仓库内参考无法发现的共同盲区错误）、在有此承诺的路线上做并行逐位一致性测试、经泛型 driver 的每 ISA 内核运行，以及——容易被低估的——安全 API 的精确 panic 措辞，由 `#[should_panic(expected = ...)]` 的子串（如 `"A.cols"`、`"aliases itself"`）把关，校验信息不会悄悄退化成没那么有用的版本。

性质测试把扫描推广开：`tests/props_api.rs`（oracle 精度、跨运行位级确定性、串行 == 并行、`beta == 0` 的覆写语义、广播步长、批量、panic 保证）、`tests/props_packed.rs`（一般情形下预打包与普通 gemm 逐位一致、文档化的 tiny/gemv 例外集上按容差）、`tests/props_knobs.rs`（随机旋钮设置下的行为），用 proptest 驱动形状、步长和旋钮值。

再往下一层，`tests/simd_conformance.rs` 检查 L0 词汇表本身：宿主支持的每个 ISA token 被直接构造——绕过分发——每个 `SimdOps` 原语、同质 `KernelSimd` blanket 实现、可移植的 `fma_bvec` 回退都逐 lane 对照标量参考。产品内核很少触碰的原语（整数 `reduce_sum`、`fnma`、拓宽接缝）就是在这里得到覆盖的；模块内扫描（如 `gemmkit/src/simd.rs` 里 `requant_store` 的逐位相等测试）对向量重量化契约做同样的事。该套件不依赖 proptest，因此也能在 wasm 上运行，对编译期的 `simd128` token 做一致性测试。

模糊测试位于 `gemmkit/fuzz/`，一个仅 nightly 的 cargo-fuzz 子工作空间（自己的 workspace 根，从稳定构建中排除），有六个 libFuzzer 目标：`fuzz_gemm`（构造即合法的问题，与朴素参考做差分检查——任何 panic 都是库缺陷）、`fuzz_batched`、`fuzz_prepack` 与 `fuzz_prepack_i8`（穿过预打包 API 的往返，i8 版按位精确把关）、`fuzz_api_validation`（把对抗性几何喂给*带校验*的入口，文档化的 `"gemmkit:"` panic 是可接受结果，其余一切都是校验缺口），以及 `fuzz_knobs`——每次运行前把所有进程级调优旋钮设成对抗值，正是它机械地找出了分块模型里的算术溢出类缺陷。

## 隔离纪律

两类全局状态会让随意的测试组织产生竞争，套件的布局正是围绕它们塑造的。

调优旋钮是进程级原子变量。每个会改动旋钮的测试都住在专属二进制里——`tests/tuning.rs`（setter）、`tests/env.rs`（环境变量解析；它只含一个测试，环境访问从构造上就是单线程的）、`tests/props_knobs.rs`、`tests/deep_k_narrow.rs`（拨动 `GEMMKIT_DEEP_KC_BYTES` 强制走各条深 K 路线）——因为独立二进制就是独立进程，无法与另一个二进制的旋钮状态竞争。*在*每个二进制内部，libtest 仍然并发跑测试，所以每个碰旋钮的测试都在该二进制的 `KNOB_LOCK` 互斥锁下串行并恢复自己改过的值（性质测试二进制另加一个能在 proptest 内部 `catch_unwind` 中存活的 RAII 守卫）。

`GEMMKIT_REQUIRE_ISA` 更黏：分发对它每进程只记忆一次。于是每个取值一个钉住二进制——`tests/env_isa_avx512f.rs`、`_vnni`、`_bf16`、`_scalar`、`_neon`、`_wasm`，外加断言未知值 panic 的 `env_isa_garbage.rs`。每个二进制的所有测试都经 `tests/env_isa_common/` 里共享的 `Once` 在任何分发解析之前完成那唯一一次 `set_var`；同一二进制内所有测试钉同一个值，谁先抢到 `Once` 无关紧要。这次写入刻意*覆盖*继承来的 `GEMMKIT_REQUIRE_ISA`——正是这一点让下文 SDE 钉住的 CI 任务能运行这些二进制，同时仍然走真实的每 ISA 路线。

Miri 补上模糊测试的 sanitizer 覆盖不到的内存安全部分：CI 在 Miri 下运行标量路径正确性套件（`miri_scalar_path`）与复数负步长 unchecked 入口，逐条解释打包与 microkernel 路径上真实的 unsafe 指针算术。`cfg(miri)` 绕行只出现在 Miri 无法解释硬件转换的地方，绝不用来跳过逻辑。

## CI 矩阵

`.github/workflows/ci.yml` 把钉住机制变成对 runner 并不真正拥有的内核的覆盖：

| 任务 | 覆盖内容 |
|---|---|
| `test` | 默认 feature，然后 `--all-features`，然后关 `parallel`；关 `std` 的四种 feature 组合构建 |
| `kernel-scalar` / `kernel-fma` | 以 `GEMMKIT_REQUIRE_ISA` 钉住每个原生可用内核跑完整套件 |
| `avx512f_test` / `avx512vnni_test` / `avx512bf16_test` | 在 Intel SDE（`sde64 -spr`）下钉住 AVX-512F、VNNI 点积、BF16 点积内核跑套件——硅片是仿真的，代码路径是真实的 |
| `kernel-neon` | 在 arm64 macOS runner 上原生跑整个工作空间，再钉住 `neon` 重跑 |
| `wasm_simd128` / `wasm_simd128_threads` | 在 wasmtime 下于 `wasm32-wasip1` 钉住 `simd128` 跑正确性与一致性；threads 任务在 `wasm32-wasip1-threads` 上跑真实的 8 路并行 |
| `no_std` | 为 `x86_64-unknown-none`、`aarch64-unknown-none`、`wasm32-unknown-unknown` 构建 |
| `i686_check` / `msrv` / `lint` / `miri` / `coverage` | 32 位检查、Rust 1.89.0 构建、fmt + clippy `-D warnings`、上述 Miri 任务，以及仅报告的 `cargo-llvm-cov`（ISA 列表钉死，覆盖率不随 runner 池摆动） |

SDE 任务比原生慢约 50 倍，这正是 `GEMMKIT_FAST_TEST` 的用武之地：它是一个**只属于测试套件**的开关——在 `tests/fast_test_common/` 实现一次、由各 harness 包含；库本身从不读它——把确定性的维度/系数扫描收缩到每个冗余组合一个代表，同时仍然走到每个分支和路径类。SDE 任务设置它（连同 `PROPTEST_CASES=16`）；原生任务保留完整扫描。把它挡在库之外，意味着测试便利开关不可能改变发布行为。

最终效果把整章收拢起来：前面各页做出的每条断言——[特殊路径](特殊路径.md)在任意 worker 数下的逐位一致、[epilogue](Epilogue融合.md) 的 gemm-再-映射等价、[扩展接缝](扩展点.md)的开闭性质、每个被钉住的内核在项目并不拥有的硬件上的正确性——都有一个可以指认的测试把关，而且那个测试所在二进制的隔离规则让结果值得信任。
